Draft-Only Output Required

App Shell Milestone (1 of 8)

This shell defines the structure for a deterministic internal drafting assistant that only accepts de-identified scenarios and only produces draft content for staff review.

No external AI calls are available in runtime behavior for this interface.

Hard Constraints

  • Do not process PHI or identifying details.
  • Do not generate legal advice or advocacy recommendations.
  • Do not automate final NORS coding decisions.
  • Label all generated content as draft-only.
  • Prefer deterministic logic and templates whenever possible.

Module Build Plan

1) App Shell

Current milestone: navigation, guardrails, and module placeholders.

2) De-identified Input Form

Controlled intake with required de-identification checks.

3) Case Note Draft Engine

Template-based drafting from JSON rules in /data.

4) Checklist Validation

Deterministic checks for required note elements.

5) Email Draft/Summary

Local, rule-based email draft and summarization workflows.

6) Local Reference Lookup

Search and surface relevant text from local source files.

7) Sample Data + Tests

Fixture scenarios plus repeatable functional checks.

8) UI Polish

Accessibility, visual consistency, and workflow refinement.

Proposed Next Step

Build milestone 2 by adding a de-identified case intake form with deterministic validation rules and explicit blocking when identifying data is detected.
